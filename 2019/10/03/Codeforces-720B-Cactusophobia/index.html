Codeforces 720B Cactusophobia

發表於 2019-10-03 | 更新於 2019-10-04 | 分類於 Codeforces | 評論數：

給你仙人掌圖 每條邊上有顏色 請你remove一些邊並同時讓剩餘顏色數最多

Solution 1

仙人掌圖只要移除所有環中的一條邊就轉成Tree了
把問題轉成flow，我們把環找出來,因為他不能全拿，如果有環的話，從source接到環，cap=size-1，然後環在連接顏色
不是環的話直接連接顏色，cap=1，最後把所有顏色連接sink，cap=1
跑dinic後的 max flow 就是ans

Solution 2

首先我們將仙人掌圖中的環視為一個點，連向所有包含於此環的顏色
因為我們一樣要扣掉環中的某條邊，但是當我們dsu找到環的時候代表有多的邊可以remove而不影響顏色數
最後每個 set size 加起來 (如果set中無環要-1) 就是答案

Code

Solution 3

二分匹配
一樣把環當一個點連邊 但只跑len-1次匹配
其他非環上的邊就存兩次 反正都只跑len-1次(2-1=1)
並查集還是比較短且快 但多次匹配的想法蠻不錯的
另外 我把非環上的邊都建在0 卻因為重複匹配的關係花的時間久很多
